# Triggering examples

Trigger text should make it obvious when an agent should be used.

## What good trigger text does

- States the exact task class
- Covers common phrasings
- Avoids vague wording
- Makes boundaries clear

## Useful guidance

1. Include the task name in plain language
2. Mention related synonyms
3. Clarify when not to use the agent
4. Keep it short enough to read quickly

## For portable agents

If a client supports example blocks, use them to show intent and scope. If it does not, fold the same information into a concise description.
